🚨 هشدار امنیتی بسیار مهم: آسیب‌پذیری بحرانی در وب‌سرور آپاچی (Apache)

⚠️ یک باگ امنیتی خطرناک با شناسه CVE-2026-23918 (با امتیاز حساسیت 8.8) در قابلیت HTTP/2 وب‌سرور Apache کشف شده است.
این آسیب‌پذیری می‌تواند منجر به اجرای کد از راه دور (RCE) و حملات DoS شود؛ به این معنی که هکرها می‌توانند از راه دور کنترل سرورهای آسیب‌پذیر را به دست بگیرند!

🛡 راه‌حل:
برای رفع این مشکل و جلوگیری از هک شدن، باید فورا آپاچی سرور خود را به نسخه 2.4.67 آپدیت کنید.

💻 دستورات آپدیت برای سیستم‌عامل‌های مختلف:

◀️ سرورهای کلادلینوکس (CloudLinux):
yum update ea-apache24 --enablerepo=cl-ea4-testing


◀️ توزیع‌های پایه RHEL (مثل CentOS):
yum clean all
yum makecache
yum -y update ea-apache*


◀️ آلمالینوکس (AlmaLinux):
dnf clean all
dnf makecache
dnf -y update ea-apache*


◀️ اوبونتو (Ubuntu):
apt update
apt install --only-upgrade "ea-apache24*"


✅ بررسی صحت آپدیت:
پس از اجرای دستورات بالا، کامند زیر را وارد کنید. خروجی باید نسخه Apache/2.4.67 را نشان دهد:
httpd -v

🛡 نرم افزار AdGuard Home:
این نرم افزار فیلترشکن نیست ولی خیلی ابزار مهم و جالبیه! در واقع یک DNS Server خانگی (و شخصی) هست که بهتون قابلیت‌های بی‌نظیری میده.

✨ چرا به دردمون می‌خوره؟
◀️ جلوگیری از DNS Hijacking: با توجه به اینکه DNS سرورها داخل ایران بعضاً دستکاری میشن، این ابزار مکمل عالی برای حل این مشکله.
◀️ حذف تبلیغات: مسدودسازی تبلیغات و ترکرها در کل شبکه بدون نیاز به نصب افزونه.
◀️ مدیریت کامل: می‌تونین روتینگ خاص برای Resolve دامنه‌ها، کش کردن با زمان دلخواه و... تنظیم کنین.
◀️ امنیت بالا: پشتیبانی از پروتکل‌های امن مثل DoH و DoT.

🚀 آموزش نصب سریع (روی لینوکس):
نصب این برنامه به شدت ساده است و فقط با یک خط کد انجام میشه! کافیه به سرور لینوکسی خودتون SSH بزنید و دستور زیر رو اجرا کنید:

curl -s -S -L https://raw.githubusercontent.com/AdguardTeam/AdGuardHome/master/scripts/install.sh | sh -s -- -v


⚙️ راه‌اندازی اولیه:
بعد از اینکه نصب تموم شد، مرورگرتون رو باز کنید و آی‌پی سرورتون رو همراه با پورت 3000 وارد کنید:
🌐 http://Server-IP:3000

در این صفحه یک ویزارد ساده براتون باز میشه. پورت‌ها رو تایید کنید، یک یوزر و پسورد بسازید و تمام! حالا شما یک DNS سرور قدرتمند و شخصی دارید. 😎

✔️ نکات جانبی:
⏺ در Settings و سپس DNS settings، میتونید مثلا مشخصا کنید، چه دامنه هایی با چه ریزالورهایی Resolve بشن، مثلا میتونین بگید دامنه های .ir با ریزاولر مثلا 217.218.127.127 ریزالو بشن، نمونه:
[/ir/]217.218.127.127
1.1.1.1
8.8.8.8

⏺ یا مثلا زمان کش شدن هارو از بخش DNS cache configuration تغییر بدید یا Optimistic caching رو روشن کنین که خودش خودکار هی کش هارو آپدیت کنه و نیاز نباشه موقع درخواست منتظر بروز شدنشون باشین.

✔️ ریزالو دامنه های خارجی با X-UI و کانفیگی که به اینترنت دسترسی داره:
🔴 اگر روی همون سرور X-UI دارید، میتونید از بخش Inbounds یک انباند جدید با مشخصات مثلا زیر اضافه کنید:
Port: 5050
Target Address: 1.1.1.1
Destination Port: 53
Network: UDP


🔴 بعد داخل Xray Config قسمت Routing Rules یک Rule جدید اضافه کنین و برای Inbound همین کانفیگ جدید و برای Outbound کانفیگی که به اینترنت دسترسی داره رو انتخاب و Save و Restart کنید.

🔴 حالا داخل تنظیمات AdGuard Home میتونید، بخش Upstream DNS servers رو به این شکل در بیارید:

[/ir/]217.218.127.127
127.0.0.1:5050


🔴 اینجوری دامنه های .ir از طریق 217.218.127.127 ریزالو میشن سایر دامنه ها از همون کانفیگتون ریزالو میشن. بجای 217.218.127.127 هم میتونید از هر ریزالور دیگه ای استفاده کنید.

✅ این برنامه هم در بخش Filters دیتابیس های مختلف جهت مسدود سازی ریزالو دامنه های خطرناک، تبلیغاتی و ... رو داره که میتونید با چند کلیک دانلود و نصب کنید.
